JEC ERP · Core data model · design for review

The backbone, designed around the four daily jobs

Everything below exists to serve what the college actually does every day. We lock this shape before writing model code — so the foundation is solid from day one.

1 Attendance · semester-wise 2 Student records 3 Faculty records 4 Internal marks · per semester + Configurable RBAC · with Super Admin

The one idea that shapes everything

Attendance and marks are per semester (Odd / Even within a year). So the Semester is the hub — every time-bound record points at it. And which class a student sits in is a yearly sticker (an Enrollment), not a fixed field — which is what makes promotion and history work.

Access & roles configurable RBAC

Roles and permissions are data, not hard-coded — the admin composes custom roles in a UI. Four are seeded (Super Admin · HOD · Faculty · Student); Super Admin is bootstrapped once (no higher role exists to create it). A role is assigned to a person and revocable — so HOD rotation never means a new account.

Custom role example — “ERP Coordinator”

Super Admin composes it as HOD-like, but with some blocked: grant Student · Attendance · Marks · Reports, leave Faculty · Subject · Assignment unchecked. “Blocked” = simply not granted (a clean allow-list — no explicit deny). Scope = PROGRAM, so a coordinator only ever sees their own program's data, just like an HOD.

The four jobs, flowing through it

Every daily action is a short path across the boxes above.

Mark attendancejob 1 · per period
Faculty opens their assignment→ Class + Subject + date + period→ tap present / absent→ PeriodAttendance · period 1 → MasterAttendance
Enter internal marksjob 4 · per assessment
Faculty picks Class + Subject→ choose IA1 / IA2 / Model→ enter each student's mark→ InternalMark rows @ active Semester
Add a studentjob 2 · any year
account: name, register no…→ pick Degree + Branch (= Program)→ optional: put in Class III·A→ Enrollment @ active year
Promote at year-endrollover
select Class II·A→ “promote to III·A”→ new Enrollment per student, next year→ last year untouched = history

In one line: a Program (Degree × Branch) holds Classes (year + letter); people belong to a Program via configurable, scoped roles; a student's Enrollment places them in a class each year; and every Attendance and Internal Mark is scoped to the active Semester.
